# 01. ACT (~52M) — FT 참조군 (⚠️ Zero-shot 아님)

> **역할**: 성능 하한 베이스라인 + 수집·학습·배포 파이프라인 전구간 검증용. "VLA가 정말 더 나은가"의 대조군.
> **중요 고지**: ACT는 사전학습 체크포인트가 없는 task-specific 모델이라 **zero-shot 시연이 원천적으로 불가능**.
> 예외적으로 **파인튜닝(학습) 후 시연**하며, 모든 산출물에 **"FT 참조군"** 라벨을 명시한다.

## 모델 개요

| 항목 | 내용 |
|---|---|
| 파라미터 | ~52M |
| 사전학습 체크포인트 | ✕ (태스크별 학습) |
| 학습 입력 | 이미지 + state (LeRobotDataset) — **언어 필드 불필요** |
| 추론 입력 | 카메라 프레임 + 현재 joint state |
| 출력 | joint action chunk (향후 k-step 관절값, action chunking) |
| 특성 | 가벼움·빠름·단일태스크 강함, 언어 조건 불가 |
| 5070 Ti 16GB | ✅ 완전 여유 (Full FT 가능) |
| SO-ARM 궁합 | SO-ARM101은 ALOHA와 같은 leader–follower·joint-space 시연 방식이라 ACT와 궁합이 좋음 |

## 데이터 요구사항

- **포맷**: LeRobotDataset 그대로, 추가 요구사항 없음
- **수집량**: T1 ~30ep + T2 ~30ep = 60ep (teleop)
- **수집 규칙**:
  - 3지점 그리드(테이핑) 커버
  - **카메라 뷰만 보고 조작** (cheating 방지)
  - 레퍼런스 데이터셋(`lerobot/svla_so101_pickplace`, `youliangtan/so101-table-cleanup`)의
    물체·배치·조명을 최대한 재현 — zero-shot 3종과 조건 통일
  - 카메라 이름은 공통 규칙(`top`/`wrist`) 준수 — 이후 4모델 재사용 전제

## 결과 기록

### T1. Pick-and-Place (FT) — 측정일 2026-07-09

측정 조건: 학습 정책 `heongyu/act_so101_t1` 직접 제어, 제어 루프 ~25Hz(30Hz 목표, chunk 경계 지연), VRAM 피크 **1.36GB**, eval 데이터셋 `heongyu/eval_act_so101_t1`.
성공/실패는 에피소드 길이로 판정(성공=조기 종료 11~17초 / 실패=60초 타임아웃). 그리드 지점·PR은 사용자 확인 후 확정.

그리드 순환: 1→2→3 반복 (시행 1=1번, 2=2번, 3=3번, 4=1번, …). 실패 2회는 큐브가 해당 그리드 십자에서 살짝 벗어난 위치였음.

**T1 성공률: 8/10 (80%)** — 실패는 그리드 2번·1번에서 각 1회. 특정 지점의 문제가 아니라 **정위치 여부**가 성패를 가름(같은 1번 지점도 시행4·7 성공 vs 시행10 실패).

**두 실패의 스턱 양상 차이** (ACT 특성 관찰):
- 시행8: 파지 실패했으나 정책이 "이미 잡았다"고 가정 → **빈 그리퍼로 place 위치까지 이동 후 배치 모션 반복**. action chunk가 파지 성공 시연을 그대로 재생.
- 시행10: 파지 실패 상태를 인지한 듯 **원위치(3번)에서 pick 모션만 무한 반복**. 관측이 안 바뀌니 같은 chunk 재생성 → 제자리 진동.
- 공통: 두 경우 모두 **회복 전략 부재**. behavior cloning이 "실패 후 재계획"을 학습하지 못한 직접 증거.

**T2 성공률: 7/10 (70%)** — 10회 전부 완료(episode_index 0~9), Hub push 완료. 추가 시행 불필요.

**T1과의 대조**: T1은 첫 물체 실패 시 그 자리에서 스턱(재시도 반복). T2는 **"안 되면 다음 물체로 전환"하는 유사-회복 행동이 관찰됨** (시행6: 실패 후 다른 물체 시도, 시행9·10: 앞 물체들 성공 후 마지막에서만 실패). 다만 전환 후에도 결국 파지 자체는 성공 못 함 — task 레벨의 진행은 있으나 **파지 실행 능력의 OOD 한계는 T1과 동일**.

### 요약 카드
- **셋업 특이사항**: T1=빨간 큐브+검은 박스(단일 pick-place) / T2=빨강·노랑·초록 큐브+검은 박스(3물체 연쇄 pick-place). 2캠(top/wrist) MJPG 필수, 3지점 그리드 고정. eval 시 리셋 시간에 leader(teleop)로 팔을 대기 자세로 복귀시켜야 정책이 학습 분포와 같은 상태에서 시작(미복귀 시 OOD로 전 에피소드 비정상 — 1차 측정 폐기 후 재측정).
- **수치 요약**: T1 SR **8/10 (80%)**, T2 SR **7/10 (70%)**, 평균 PR T1 ~0.85 / T2 ~0.83, 제어 루프 ~25Hz, VRAM 피크 T1 1.36GB / T2 1.45GB, 학습 loss T1 0.036(61epoch) / T2 0.052(24epoch)
- **관찰 메모**:
  - 물체가 그리드 십자에 **정확히** 놓이면 T1·T2 모두 안정적으로 성공.
  - 십자에서 조금이라도 벗어나면 **오파지** 발생 → 스턱. T1은 그 자리에서 무한 반복(재시도 루프 or 빈손 재생), T2는 **"실패한 물체를 포기하고 다음 물체로 전환"하려는 유사-회복 행동**이 관찰됐으나 전환 후에도 결국 파지 자체는 실패.
  - T2에서 앞 물체(1~2개)는 성공하고 **마지막 물체에서만** 실패하는 패턴이 다수(시행9·10) — 후반부 피로/누적 오차보다는 해당 물체의 그리드 이탈이 원인으로 추정.
  - 근본 원인 공통: (1) 학습 30ep이 전부 그리드 정위치라 위치 변동이 OOD, (2) behavior cloning이라 회복/재시도 전략을 명시적으로 학습 못 함, (3) OOD에서 action chunk가 상태를 못 바꿔 한 점에 갇힘.
- **다음 단계 시사점**: **ACT는 특정 task·고정 조건에 최적화된 소량 데이터로는 정밀도는 높지만 일반화 폭이 좁다** — "더 다양한 작업을 수행하려면 더 다양한(위치·조건) 데이터셋이 필수"라는 점을 이번 실험이 명확히 시사함. zero-shot VLA 3종(SmolVLA/π0/GR00T)은 대규모 다양한 사전학습 데이터를 갖고 있어 이 위치 강건성 축에서 ACT 대비 어떤 차이를 보이는지가 핵심 비교 포인트. 향후 ACT 개선 실험을 한다면 그리드 정위치뿐 아니라 **연속적 위치 변동 + 파지 실패→재시도 시연**을 포함해 robustness 향상 여부를 검증할 수 있음.
