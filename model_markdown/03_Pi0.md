# 03. π0 (3B) — Zero-shot + 원격 5090 FT 경로

> **역할**: 가장 강한 일반화 계열의 대표. 로컬 5070 Ti에서는 **zero-shot 추론 비교** 담당,
> 파인튜닝은 **공용 RTX 5090(32GB) 원격 Docker LoRA FT → 체크포인트 회수** 경로로 확정.
> π0 vs π0.5: LeRobot 내 레퍼런스가 많은 **π0로 시작** (문제 발생 시 참고 사례 풍부).

## 모델 개요

| 항목 | 내용 |
|---|---|
| 파라미터 | ~3B |
| 사전학습 체크포인트 | ✅ LeRobot 포팅 체크포인트 |
| 사전학습 데이터 | OXE + 자체 멀티임바디먼트 — SO-ARM 계열은 native 아님 (FT 전제) |
| 학습 입력 | 이미지 + state + 언어 (LeRobotDataset + norm stats) |
| 추론 입력 | 카메라 프레임 + joint state + 자연어 명령 |
| 출력 | flow-matching action chunk (고주파, ≈50Hz) |
| VRAM 요구 | 추론 >8GB (bf16) / LoRA FT >22.5GB / Full FT >70GB |
| 5070 Ti 16GB | 추론 ✅ (bf16) / FT ✕ 로컬 불가 |
| RTX 5090 32GB | LoRA FT ✅ / Full FT ✕ |

## 데이터 요구사항 (향후 FT 대비)

- **포맷**: LeRobotDataset + **normalization 통계(norm stats)**, `meta/info.json` 스키마 정합
- v2.1은 `episodes_stats.jsonl` 필요 — 변환 스크립트 제공 (난이도 낮음)

## 원격 파인튜닝 워크플로 (확정 경로, 향후 FT 단계)

```
[5070 Ti PC]                         [공용 5090 서버]
데이터 수집(teleop) ──push──▶ HF Hub private repo ──pull──▶ Docker에서 lerobot-train (π0 LoRA)
로컬 추론 배포(bf16)  ◀──pull── HF Hub 체크포인트 repo ◀──push── 학습 완료 체크포인트
```

- **Docker 이미지**: CUDA 12.8+ base + 최신 PyTorch + `lerobot[pi0]` — 5090도 동일 Blackwell(sm_120)이라 5070 Ti와 이미지 재사용 가능
- **공용 장비 예절**: `docker run --gpus '"device=0"'` GPU 지정, 실행 전 `nvidia-smi` 점유 확인, 체크포인트 주기 짧게(resume 대비) + 시간 예약
- **체크포인트 회수 후**: LoRA adapter를 base에 **merge해서 배포** (재현 변수 감소). 추론은 bf16으로 16GB 수용 — 해상도·배치는 보수적으로
