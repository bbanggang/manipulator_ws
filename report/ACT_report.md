# ACT 모델 진행 보고서 — SO-ARM101 (T1 Pick-and-Place / T2 Table Cleanup)

> 작성일: 2026-07-09 | ACT-T1·T2 학습·평가 완료, **ACT 모델 완결**
> 프로젝트: 단일 SO-ARM101 매니퓰레이터로 4개 모델(ACT / SmolVLA / π0 / GR00T N1.5)의
> 학습 및 zero-shot 성능을 비교한다. 본 문서는 그 첫 모델인 **ACT**의 전 과정 요약이다.
> 나머지 3개 모델은 **동일한 T1·T2 task, 동일한 물리적 셋업**(그리드 3지점·카메라·물체)으로
> zero-shot 추론을 수행해 "학습(FT) vs 사전학습만(ZS)"을 공정 비교한다.

---

## 1. 개요 — 무엇을, 왜 했는가

4개 비교 모델 중 **ACT를 가장 먼저** 다뤘다. ACT는 사전학습 체크포인트가 없어
zero-shot(학습 없이 바로 추론)이 불가능한 모델이라, "데이터 수집 → 학습 → 추론 평가"의
전체 파이프라인을 직접 거쳐야 한다. 이를 먼저 완주함으로써:

- **파이프라인 검증**: 이후 3개 모델이 공용으로 쓸 수집·평가 절차를 미리 확립
- **기준선 확보**: "소량 데이터로 파인튜닝하면 이 정도"라는 성능 참조점 (zero-shot 모델들과 대비할 하한/상한 기준)
- **데이터 자산화**: 여기서 수집한 시연 데이터는 이후 SmolVLA/π0/GR00T 파인튜닝에 재사용

**Task 2종** (4모델 공용):
- **T1. Pick-and-Place**: 빨간 큐브를 집어 검은 박스에 넣는다.
- **T2. Table Cleanup**: 빨강·노랑·초록 큐브 3개를 순차로 집어 모두 검은 박스에 넣는다 (한 에피소드에 3회 연쇄 pick-place).

---

## 2. 실험 셋업

### 2.1 로봇과 카메라

- **로봇**: SO-ARM101 leader–follower 한 쌍. 사람이 leader를 움직이면 follower가
  실시간으로 따라 하며(teleoperation), 이 궤적을 시연 데이터로 기록한다.
- **카메라 2대** (상호 보완):
  - `top` — 작업 공간을 위에서 내려다보는 뷰. "물체·박스가 어디 있고 어디로 가야 하는가"(전역 위치)
  - `wrist` — 그리퍼에 달린 1인칭 뷰. "지금 이 각도로 잡을 수 있는가"(국소 정밀)
  - 두 뷰를 모두 모델 입력으로 사용한다. 640×480 @ 30fps, MJPG(대역폭 문제로 필수).

### 2.2 데이터셋

| | T1 (Pick-and-Place) | T2 (Table Cleanup) |
|---|---|---|
| 물체 | 빨간 큐브 1개 | 빨강·노랑·초록 큐브 3개 |
| 에피소드 | 30개 | 30개 |
| 프레임 | 13,032 (~7.2분) | 32,835 (~18.3분) |
| Hub | `heongyu/so101_t1_pickplace` | `heongyu/so101_t2_cleanup` |

teleop으로 수집, 물체 시작 위치를 테이프로 표시한 **3지점 그리드**에 순환 배치해 위치
다양성을 일부 확보했다(T2는 색 배치도 매 에피소드 셔플). 각 프레임에는 두 카메라 영상 +
follower의 관절 상태(6-DOF) + 목표 관절값(action)이 동기화되어 저장된다 (LeRobotDataset 포맷).

---

## 3. 학습 과정

### 3.1 ACT의 작동 원리 (핵심만)

ACT(Action Chunking with Transformers)는 **사람의 시연을 그대로 모방**하는
행동 복제(behavior cloning) 모델이다. 두 가지 특징이 핵심이다:

- **Action Chunking**: 매 순간 1스텝씩이 아니라 **미래 100스텝(약 3.3초)의 관절 궤적을
  한 번에 예측**한다. 덕분에 동작이 매끄럽고, 오차가 누적되는 문제에 강하다.
- **Transformer + CVAE 구조**: 이미지·관절 상태를 입력받아 궤적을 생성하며,
  사람 시연의 스타일 변동을 잠재변수로 압축해 학습한다.

### 3.2 학습 설정과 결과

| 항목 | T1 | T2 |
|---|---|---|
| 모델 크기 | 51.6M 파라미터 (경량, 공통) | 〃 |
| 학습량 | 100,000 step / **61.4 epoch** | 100,000 step / **24.4 epoch** (데이터 2.5배라 epoch 낮음) |
| 소요 시간 | ~3시간 (RTX 5070 Ti) | ~3시간 |
| Loss 함수 | L1 loss(궤적 재현 오차) + 10×KL 정규화 (공통) | 〃 |
| 최종 loss | 6.74 → **0.036** | 6.71 → **0.052** |
| 학습 VRAM | 5.3GB / 16GB | 5.3GB / 16GB |
| 체크포인트 | `heongyu/act_so101_t1` | `heongyu/act_so101_t2` |

T2가 T1보다 loss가 다소 높은 것은 (a) epoch 수가 적고 (b) task 자체가 3배 복잡(연쇄
pick-place)하기 때문으로, 예상된 정상 범위다.

## 4. 추론(평가) 과정

학습된 정책이 **follower를 직접 제어**하도록 하여 실제 로봇으로 task를 수행시켰다.
매 순간 두 카메라 영상과 관절 상태를 정책에 입력 → 정책이 다음 동작(관절값)을 출력 →
로봇이 실행하는 폐루프(closed-loop) 방식이다.

- **평가 프로토콜**: 3지점 그리드를 순환하며 **각 task 10회 시행**, 개입 없이 관찰
- **추론 성능**: 제어 루프 ~25Hz, 추론 VRAM T1 1.36GB / T2 1.45GB (학습 대비 매우 가벼움 — batch=1)
- 실무 이슈: 리셋 시간에 leader로 팔을 대기 자세로 되돌려야 정책이 학습 분포와 같은
  상태에서 시작한다 (미복귀 시 비정상 시작 → 1차 측정 폐기 후 재측정)

## 6. 실패 분석

두 task 실패 모두 **파지 실패(오파지)로 시작**했고, 이후 한 동작에 갇혔다(스턱). 다만
task 구조 차이로 스턱 양상에 흥미로운 차이가 있었다:

| Task | 스턱 양상 | 해석 |
|---|---|---|
| T1 | 빈 그리퍼로 배치 동작 반복 / 파지 동작만 무한 반복 | 정책이 관측을 무시하고 학습된 궤적을 그대로 재생, 또는 실패 상태에 갇혀 같은 궤적 재생성 |
| T2 | **실패한 물체를 포기하고 다음 물체로 전환**하려는 시도(시행6) / 앞 물체 성공 후 마지막에서만 실패(시행9·10) | task 레벨에서는 "다음 단계로 진행"하는 유사-회복 행동이 나타나지만, **파지 자체의 성공 여부는 여전히 위치에 의존** — 전환해도 결국 그 물체를 못 잡으면 실패로 끝남 |

**근본 원인 3가지** (공통):

1. **분포 밖(OOD) 상태**: 학습 데이터가 전부 그리드 정위치라, 위치가 조금만
   벗어나도 모델이 한 번도 본 적 없는 입력이 된다.
2. **회복 전략 부재**: ACT는 시연을 모방할 뿐 "파지 실패했으니 다시 시도"를 명시적으로
   배운 적이 없다. T2에서 관찰된 "다음 물체로 전환"은 task 순서상의 진행이지, 파지
   실패를 인식하고 재시도하는 능력은 아니다.
3. **개루프적 재생**: 예측한 100스텝 궤적이 상태를 못 바꾸면 다음 입력도 비슷해
   같은 궤적을 반복 → 무한 루프.

> 이는 결함이 아니라 **ACT + 소량·고정조건 데이터의 알려진 특성**이며, 오히려 이
> 프로젝트가 ACT를 "참조군"으로 둔 이유를 정확히 입증하는 결과다.

## 7. 결론

- ACT는 **정밀 배치엔 강하지만(정위치 T1 80%·T2 70%) 위치 변동엔 취약**하다. T2처럼
  task가 복잡해져도 핵심 병목은 동일하게 "파지 시점의 위치 정합"이었다.
- 51.6M 경량 모델이 task당 3시간 학습·1.5GB 미만 추론으로 저사양 GPU에서 완결 가능함을 확인.
- **"더 다양한 작업을 수행하려면 더 다양한(위치·조건) 데이터셋이 필수"** — 이번 실험이
  가장 명확하게 시사하는 결론이다. 그리드 3지점처럼 좁고 고정된 조건의 소량 데이터로는,
  같은 조건 안에서는 높은 정밀도를 내지만 그 바깥으로는 전혀 일반화되지 않는다.
- 이 "위치 강건성(robustness)"이 이제 **zero-shot VLA 3종(SmolVLA/π0/GR00T)과 대비할
  핵심 비교 축**이 된다 — 대규모·다양한 데이터로 사전학습된 VLA가 같은 그리드 이탈
  상황에서 ACT보다 안정적으로 대응하는지가 다음 검증 대상.
- 개선 방향(향후 실험 후보): 그리드 정위치뿐 아니라 **연속적 위치 변동 + 파지
  실패→재시도 시연**을 포함해 ACT의 robustness가 실제로 개선되는지 검증.

  정량·정성적으로 정리하는 것.
